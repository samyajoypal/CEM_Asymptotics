# Boundary-aware inference for classification likelihood

Reproducibility repository for *Inference for Hard Classification
Likelihoods: Moving-Boundary Curvature and Feasible Uncertainty
Quantification*. The public repository is
<https://github.com/samyajoypal/CEM_Asymptotics>.

This is the independent research and reproducibility project for the proposed
JRSS Series B paper.  The earlier JSTA manuscript, code, and results in the
parent directory are retained unchanged.

## Scientific objective

Develop general, scalable inference for classification maximum-likelihood
estimators by estimating the curvature generated by active decision
boundaries.  The new project is not restricted to two-dimensional integration
grids or to a particular pair of component families.

## Project layout

- `paper/`: new manuscript and bibliography; `paper/legacy/` is reference-only.
- `src/cem_inference/`: installable statistical implementation.
- `tests/`: mathematical and numerical validation tests.
- `scripts/`: reproducible simulation and data-analysis entry points.
- `configs/`: versioned experiment specifications.
- `hpc/`: scheduler scripts and run manifests.
- `data/`: documented raw and processed data; large/private data are excluded.
- `results/`: generated outputs; only compact final artefacts should be tracked.
- `docs/`: theory audits, design decisions, and reproducibility documentation.

## Development order

1. Freeze the estimand, parameterization, and active-boundary notation.
2. Establish population differentiability and asymptotic normality.
3. Construct and prove consistency of a feasible boundary-curvature estimator.
4. Implement distribution adapters and verify density/score/Hessian identities.
5. Validate on small deterministic experiments before Monte Carlo work.
6. Run the redesigned simulations and real-data analyses.
7. Finalize the main paper, supplement, and public reproducibility archive.

No result from the legacy project is treated as evidence in the new paper
until it is regenerated and validated by this project.

## Local setup

The parent project's `.venv` currently contains FMVMM 3.0.0 and can be used
during initial development:

```bash
source ../.venv/bin/activate
python -m pip install -e .
python -m pytest
```

The initial implementation now includes exact Normal, Student-$t$, and
multivariate skew-normal adapters; shared unconstrained coordinates; numerical
score/Hessian validation; active-pair detection; the kernel tube estimator;
and sandwich assembly. Run the validation suite with:

```bash
../.venv/bin/python -m pytest -q --cov=cem_inference
```

## Rebuild the submission

The compact processed results required by the paper are versioned. The 3.2 GB
replication-level HPC archive is intentionally excluded; its deterministic
reduction is versioned as
`results/processed/confirmatory_v2/normality_diagnostics.csv`, and can be
regenerated with `scripts/summarize_normality.py` when the raw archive is
available.

```bash
bash scripts/build_submission.sh
../.venv/bin/python -m pytest -q
```

This regenerates every paper figure and table and builds the main manuscript
and supplement with the official OUP/RSS class. Scheduler manifests and
processing scripts are retained under `hpc/` and `scripts/`.
